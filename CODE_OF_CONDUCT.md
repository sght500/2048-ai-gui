# Code of Conduct

## Our Pledge

We are committed to providing a welcoming, respectful, and productive environment for everyone contributing to this project.

This project exists to explore **human-in-the-loop AI**, usability, and cross-platform tooling. Collaboration, curiosity, and constructive feedback are core to its success.

By participating, you agree to uphold the principles outlined below.

---

## Expected Behavior

Contributors are expected to:

* Be respectful and considerate in discussions, issues, and pull requests.
* Focus on **technical merit**, clarity, and shared learning.
* Provide constructive feedback and accept feedback gracefully.
* Assume good intent from other contributors.
* Keep discussions relevant to the project goals (GUI, AI interaction, usability, cross-platform concerns).

---

## Unacceptable Behavior

The following behaviors are not acceptable:

* Harassment, insults, or discriminatory language of any kind.
* Personal attacks, trolling, or hostile communication.
* Dismissing contributions without technical justification.
* Repeatedly derailing discussions away from the project’s scope.
* Any behavior that would reasonably make others feel unwelcome or unsafe.

---

## Scope

This Code of Conduct applies to:

* GitHub issues
* Pull requests
* Code reviews
* Discussions and comments
* Any project-related communication spaces

---

## Enforcement

Project maintainers are responsible for clarifying standards and taking appropriate action if these guidelines are violated.

Actions may include:

* Requesting clarification or adjustment of behavior
* Moderating or closing discussions
* Rejecting pull requests that violate project standards

Maintainers will act in good faith and with proportionality.

---

## Reporting Concerns

If you experience or witness unacceptable behavior, please report it by opening a private issue or contacting the repository maintainer.

All reports will be handled discreetly and respectfully.

---

## Final Note

This is a **learning-friendly, experimentation-driven project**.
Kindness, patience, and clear communication help everyone build better tools—and better games.

Thank you for being part of this project.
